Issue Tracker

The GeoTools issue tracker is used to report problems and capture tasks and improvements.

To report a problem:

  1. Check the open issues list for an existing report, and add a comment with your report.
  2. If your issue has not yet been reported please add a new issue. Note the version of GeoTools and the steps required to reproduce the issue.
  3. If your issue is critical, with a simple fix, please attach a patch. You may wish to raise it on geotools-devel for review.
  4. If your issue is critical but difficult to resolve, you may consider the commercial support options on this page.

Documentation

Programmers wishing to use GeoTools in their own applications can get more information from the User Guide. The User Guide contains a welcome section to get you started using a series of tutorials, along with a reference section and code examples for each module in the library.

Developers wishing to extend the GeoTools library can start out with the Developer Guide. The Developers guide captures our build instructions, policies and procedures and the care and feeding instructions for the library as a whole.
